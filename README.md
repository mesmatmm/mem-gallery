# MEM Gallery

A modern, minimalist photography portfolio website showcasing 755+ high-quality optimized photos with automatic gallery generation, interactive filtering, dark/light theme toggle, responsive design, and a secure admin dashboard for easy management.

## 🌐 Live Demo

**[Visit MEM Gallery →](https://mesmatmm.github.io/mem-gallery/)**

---

## ✨ Features

- **🎨 Auto-Updating Gallery** - Add images to folders, run optimize + generate, they appear in the gallery
- **🔐 Admin Dashboard** - Secure login-protected control panel to manage gallery operations
- **📸 Smart Image Optimization** - WebP conversion with manifest tracking to skip already-processed files
- **🗑️ Orphan Sync** - Automatically removes optimized/thumbnail files when originals are deleted
- **✨ Sophisticated Animations** - Fade-in, slide, and hover effects on images
- **Responsive Design** - Mobile-first approach with full tablet and desktop support
- **Dark/Light Theme Toggle** - User preference saved to localStorage
- **Gallery Filtering** - Filter images by folder/category
- **Real-time Search** - Find images by folder name, title, caption, or tags
- **Tag-Based Organization** - Automatic tagging system for better discoverability
- **Lightbox Viewer** - Fullscreen image viewing with keyboard and touch navigation
- **Featured Section** - Random images with animations on home page
- **Live Stats Strip** - Animated count-up counters showing total photos and collections
- **Lazy Loading** - Native image lazy loading for better performance
- **Accessible** - Semantic HTML, ARIA labels, keyboard navigation (arrows, ESC, Tab)
- **SEO Optimized** - Meta tags, structured data (JSON-LD), Open Graph, Twitter Cards
- **No Framework Dependencies** - Vanilla HTML5, CSS3, and JavaScript

## 🚀 Quick Start

### Installation

```bash
git clone https://github.com/mesmatmm/mem-gallery.git
cd mem-gallery
npm install
```

### Run Locally

#### Admin Dashboard (Recommended)

```bash
npm run admin
# Opens http://localhost:3001
# Login with credentials configured in .admin-config.json
```

#### View Gallery Only

Open `index.html` directly, or use any local server:

```bash
npx http-server
# Visit http://localhost:8080
```

#### Manual Scripts

```bash
npm run optimize    # Sync orphans + compress originals to WebP (skips unchanged files)
npm run generate    # Regenerate gallery.js from images-optimized/
```

## 🔐 Admin Dashboard

A secure admin panel at `http://localhost:3001` for managing photos without the command line.

### Features

- **Optimize Images** - Syncs orphaned files, converts new originals to WebP, skips already-processed ones. Reports: processed / skipped / removed counts.
- **Regenerate Gallery** - Rebuilds `gallery.js` from the current state of `images-optimized/`
- **Edit Entries** - Update captions, categories, and tags per photo
- **Delete Entries** - Remove a gallery entry from `gallery.js`
- **View Stats** - Folder breakdown, total image counts, optimization status

### Default Credentials

Stored in `.admin-config.json` (auto-created, gitignored):

```json
{ "username": "Admin", "password": "changeme" }
```

Change credentials via the Settings panel in the dashboard.

## 📸 Adding Images

### Workflow

1. **Add folder and images to `assets/images/`**

   ```text
   assets/images/MyNewFolder/
   ├── photo1.jpg
   └── photo2.png
   ```

2. **Optimize** (converts to WebP, creates thumbnails, skips already-done files)

   ```bash
   npm run optimize
   ```

3. **Regenerate gallery.js**

   ```bash
   npm run generate
   ```

4. **Commit and push** — GitHub Pages will serve the updated site.

### What Happens Automatically

- ✅ Orphaned optimized/thumbnail files removed (if their originals were deleted)
- ✅ New images converted to WebP (optimized at 1920px + thumbnails at 480px)
- ✅ Already-processed, unchanged files skipped via manifest
- ✅ `gallery.js` regenerated with all current images
- ✅ Existing captions, tags, and categories preserved

---

## 🔧 How Optimization Works

`optimize-images.js` runs in two phases every time:

**Phase 1 — Sync orphans**
Scans `images-optimized/` and `images-thumbs/` for files whose original no longer exists in `images/`. Deletes those files and cleans up empty folders. Also removes their entries from the manifest.

**Phase 2 — Process originals**
Scans `images/` for new or modified files (compared by `mtime` in `.optimization-manifest.json`). For each changed file, creates a WebP optimized version and a WebP thumbnail. Updates the manifest entry.

**Result:** `images-optimized/` and `images-thumbs/` always mirror exactly what is in `images/`.

## 🌐 Deployment (GitHub Pages)

1. Push to `main` branch
2. Go to repository Settings → Pages → Source: `main / (root)`
3. Visit `https://yourusername.github.io/mem-gallery/`

> **Important:** The `.nojekyll` file in the root is required. Without it, GitHub Pages runs Jekyll which silently drops any file or folder whose name contains spaces — causing images to return 404.

## 🎨 Customization

### Colors

Edit `:root` variables in `assets/css/main.css`:

```css
:root {
    --color-bg: #0f0f0f;
    --color-accent: #c9a96e;
    --color-accent-hover: #b8935a;
}
```

### Category & Tag Mapping

Edit `FOLDER_CATEGORIES` and `FOLDER_TAGS` in `generate-gallery-data.js`:

```js
const FOLDER_CATEGORIES = {
    'MyNewFolder': 'landscape',
    ...
};
const FOLDER_TAGS = {
    'MyNewFolder': ['tag1', 'tag2'],
    ...
};
```

Then run `npm run generate`.
